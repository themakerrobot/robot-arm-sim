# 로봇팔 시뮬레이터 (Robot Arm Simulator)

웹에서 4축 로봇팔을 시뮬레이션하고, 그 값을 **실물 로봇팔(아두이노)** 로 보내 똑같이 움직이는 교육용 도구입니다.

**실행 주소:** https://themakerrobot.github.io/robot-arm-sim/

브라우저는 **Chrome 또는 Edge** 를 쓰세요 (실물 연결 기능 Web Serial이 이 두 곳에서만 동작합니다). 페이지를 열면 `data/` 폴더의 모델(arm4.urdf + STL)이 자동으로 불러와집니다.

---

## 1. 화면 구성

| 영역 | 위치 | 역할 |
|---|---|---|
| 3D 뷰 | 가운데 | 로봇팔이 보이는 곳 |
| 관절 패널 | 오른쪽 | 관절별 슬라이더로 자세 조절 |
| 시리얼 패널 | 왼쪽 위 | 실물 연결·전송·보정 |
| 타임라인 | 아래 | 키프레임으로 모션(동작) 제작 |

**3D 뷰 조작:** 좌클릭 드래그=회전, 우클릭 드래그=이동, 휠=확대/축소.

---

## 2. 관절 움직이기

오른쪽 관절 패널에서 각 관절을 조절합니다.

- **슬라이더 값은 그대로 서보 모터값(0~180)** 입니다. `90 = 중립(직립)`.
- 슬라이더를 드래그하거나, 값 양옆의 **`−` / `+` 버튼으로 1도씩** 미세조정.
- `−`/`+` 를 **길게 누르면 연속**으로 움직입니다.
- 그리퍼는 **0 = 열림(간격 45mm), 180 = 닫힘(간격 15mm)** — 값 칸에 죠 간격이 mm로 표시됩니다.

왼쪽 **포즈 → 리셋** 을 누르면 중립(모든 관절 90, 그리퍼 0)으로 돌아갑니다.

---

## 3. 실물 로봇팔 연결 (Web Serial)

왼쪽 위 **실물 연결(Serial)** 영역에서 진행합니다.

1. **Baud 를 `9600`** 으로 둡니다. (아두이노 펌웨어가 9600이라 반드시 맞아야 함)
2. **연결(Connect)** 클릭 → 브라우저 포트 선택창이 뜨면 로봇팔의 **COM 포트를 선택**.
   - Web Serial은 보안상 COM 목록을 미리 못 보여줍니다. 이 선택창이 COM을 고르는 자리입니다.
3. 연결되면 아래에 `USB VID:.. PID:.. · 9600baud` 가 표시되고, 로그창에 펌웨어의 `Booting... / Ready` 가 깨끗한 글자로 떠야 정상입니다.
4. 한 번 연결을 허용하면 **다음부터 페이지를 열 때 자동 재연결** 됩니다.

### 전송 버튼

| 버튼 | 동작 |
|---|---|
| **실시간(Live)** | 켜면 슬라이더·타임라인 값이 자동으로 계속 전송됨 |
| **현재 포즈 전송** | 지금 자세를 한 번 전송 |
| **정지 (#stop)** | 로봇팔 정지 명령 |
| **상태 (#pos)** | 현재 각도 요청 (로그에 응답이 옴) |
| **직접 명령** | `#pos!` 처럼 임의 명령을 직접 입력해 전송 |

전송 형식은 `#m:관절1,관절2,관절3,그리퍼!` 입니다. 예: 슬라이더가 90,90,90,0 이면 `#m:90,90,90,0!`.

> 명령은 가는데 안 움직이면 → **① Baud가 9600인지** ② 로그에 펌웨어 글자가 깨졌는지(=Baud 불일치) ③ 서보 전원이 충분한지 확인하세요.

---

## 4. 영점 보정 (Calibrate)

서보 혼/조립 오차 때문에 **슬라이더 90인데 실물이 정확히 직립이 아닐 때** 사용합니다. 시뮬은 그대로 두고 **실물로 나가는 값만** 보정합니다.

1. 실물을 연결하고 **실시간(Live)** 을 켭니다.
2. **영점 보정(Calibrate)** 을 켭니다. (값 칸이 초록색이 되고 옆에 `(+5)` 같은 보정값이 작게 표시됨)
3. 각 관절의 `−`/`+` 로 **실물이 일자가 될 때까지** 1도씩 맞춥니다. 누를 때마다 실물에 즉시 반영됩니다.
4. 보정값은 **브라우저에 자동 저장**되어 새로고침해도 유지됩니다. **보정값 초기화** 버튼으로 되돌릴 수 있습니다.

보정 중에도 **각도 값은 그대로 보이므로**, 보정하면서 바로 그 자세로 테스트할 수 있습니다. 보정을 끄면 `−`/`+` 는 다시 일반 각도 조정으로 돌아갑니다.

---

## 5. 모션(동작) 만들기 — 타임라인

아래 타임라인에서 여러 자세를 시간 순서로 이어 동작을 만듭니다.

1. 관절을 원하는 자세로 맞춥니다.
2. 타임라인의 원하는 시점(1초 단위)을 클릭한 뒤 **+ 키프레임** 으로 그 자세를 저장합니다. (타임라인 더블클릭으로도 추가)
3. 시점을 바꿔가며 자세를 여러 개 저장합니다.
4. **재생(▶)** 으로 동작을 확인합니다. **루프** 를 켜면 반복 재생됩니다.
5. 실시간(Live)이 켜져 있으면 재생되는 동안 **실물도 같이 움직입니다.**

- 키프레임은 **1초 단위**로 스냅되고, 기본 길이는 30초입니다.
- 키프레임을 드래그해 시점을 옮기거나, 선택 후 **삭제** 할 수 있습니다.

---

## 6. 모션 저장 / 불러오기

**저장** 을 누르면 `motion.txt` 로 **명령 텍스트**가 나옵니다:

```
m:90,90,90,0
sleep 2
m:120,60,90,30
sleep 1.5
m:90,90,90,0
```

- `m:` 줄은 그 시점의 모터값(보정 포함), `sleep` 은 다음 동작까지의 시간(초).
- **불러오기** 로 이 텍스트를 다시 읽어 타임라인으로 복원할 수 있습니다. (`#m:..!` 형식도 인식)

---

## 7. 모델 바꾸기 (data/ 폴더)

저장소의 `data/` 폴더 안 파일을 바꾸면 됩니다.

- `arm4.urdf` — 관절 구조·회전축·치수 정의
- `*.stl` — 부품 형상 (base_link, shoulder, link1, link2, moving_jaw)

STL은 **조립 위치(월드좌표) 그대로** 내보내야 합니다. URDF가 그 좌표를 기준으로 회전축을 잡으므로, 모든 관절 0(서보 90)에서 **CAD 조립 자세 그대로** 나옵니다.

> 파일을 바꿔도 화면이 그대로면 **브라우저 캐시** 때문입니다. **강력 새로고침**(`Ctrl+Shift+R`, 맥 `Cmd+Shift+R`)을 하거나 잠시 기다리세요. (이 시뮬은 캐시를 무시하도록 만들어 두었습니다.)

---

## 8. 자주 막히는 부분

| 증상 | 원인 / 해결 |
|---|---|
| 연결은 됐는데 안 움직임 | Baud가 9600이 아님 → 9600으로 재연결. 로그 글자가 깨지면 Baud 불일치 |
| 연결 때 한 번만 움직이고 멈춤 | 그건 포트 열림 때 보드가 리셋되며 홈으로 간 것(정상). 이후엔 Live를 켜고 슬라이더를 움직여야 함 |
| 명령은 보이는데 실물 정지 | 서보 전원 부족(특히 어깨 모터) → 외부 5V 전원 사용 |
| 관절이 반대로 돌아감 | `data/arm4.urdf` 해당 joint의 `axis` 부호를 뒤집기 (`0 1 0` ↔ `0 -1 0`) |
| 변경이 반영 안 됨 | 강력 새로고침. GitHub Pages는 반영에 1~2분 걸림 |
| 90에서 실물이 안 일자 | 조립 오차 → **영점 보정** 으로 맞춤 |

## 한눈에 보는 작업 순서

1. Chrome으로 사이트 열기 → 모델 자동 로드
2. **연결(9600)** → COM 선택
3. **영점 보정** 으로 실물 직립 맞추기 (처음 한 번)
4. 슬라이더/타임라인으로 동작 만들기 → **재생**
5. **저장** 으로 `motion.txt` 받기
